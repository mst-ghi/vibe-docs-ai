# Commands

Every command you need to develop, check, build, and deploy VibeDocs AI. For what the
packages behind them do see [PACKAGES.md](./PACKAGES.md); for how the project is structured
see [ARCHITECTURE.md](./ARCHITECTURE.md).

Prerequisites: **Node 20+** and **pnpm 11+** (`corepack enable` will provision the pinned
`pnpm@11.5.0`).

## npm-scripts (defined in `package.json`)

| Command          | Runs               | What it does                                                       |
| ---------------- | ------------------ | ----------------------------------------------------------------- |
| `pnpm dev`       | `next dev`         | Dev server with HMR at <http://localhost:3000>.                   |
| `pnpm build`     | `next build`       | Type-checks and statically exports the site to `out/`.            |
| `pnpm start`     | `next start`       | Serves a non-export build (rarely used here — we deploy `out/`).  |
| `pnpm lint`      | `biome check .`    | Lint + format check. **Must pass before every commit.**           |
| `pnpm lint:fix`  | `biome check --write .` | Auto-fix lint issues and format in place.                    |
| `pnpm format`    | `biome format --write .` | Format only (no lint).                                      |

## Everyday workflow

```bash
pnpm install          # install dependencies (first time / after a pull)
pnpm dev              # develop at localhost:3000
pnpm lint:fix         # tidy up before committing
pnpm build            # verify the static export succeeds
```

A change is "done" only when **both** `pnpm lint` and `pnpm build` pass.

## Dependencies (pnpm)

```bash
pnpm install              # install from the lockfile
pnpm add <pkg>            # add a runtime dependency
pnpm add -D <pkg>         # add a dev dependency
pnpm remove <pkg>         # remove a dependency
pnpm up <pkg>             # update within the semver range
pnpm up --latest <pkg>    # bump to the newest version
pnpm list                 # show the resolved dependency tree
```

## Quality gates (Biome)

```bash
pnpm lint                          # check everything (lint + format)
pnpm lint:fix                      # auto-fix what's safe
pnpm exec biome check src/app      # check a subdirectory only
pnpm exec biome check --write --unsafe .   # apply unsafe fixes (review the diff!)
```

Biome config is `biome.json`. It ignores `*.css` (Tailwind at-rules) and `next-env.d.ts`
(generated by Next).

## Build, preview, and serve the static export

```bash
pnpm build                         # → ./out (fully static)
npx serve out                      # preview the export on a local static server
python3 -m http.server -d out 8080 # alternative static preview
```

The export is just files; open `out/index.html` through any static server (opening it via
`file://` won't work because of absolute asset paths).

## Deployment (Vercel)

`vercel.json` already pins `buildCommand: pnpm build` and `outputDirectory: out`.

```bash
vercel            # deploy a preview
vercel --prod     # deploy to production
```

Or connect the Git repository in the Vercel dashboard and it will pick up `vercel.json`
automatically. Any other static host works too — point it at the `out/` directory.

## Type checking on its own

```bash
pnpm exec tsc --noEmit    # type-check without building (CI-friendly)
```

`pnpm build` already type-checks, so this is only needed for a faster standalone check.

## Useful one-offs

```bash
# Verify a lucide icon exists before importing it (this fork lacks brand icons)
node -e "console.log('Repeat' in require('lucide-react'))"

# Clean caches/artifacts and rebuild from scratch
rm -rf .next out && pnpm build
```
